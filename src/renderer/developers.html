ЯЧат для разработчиков Вернуться в ЯЧат
Digital ID API · v1.1

Подтверждённый пользователь ЯЧата в вашем сервисе

Пользователь вводит цифровой ID и подтверждает вход кодом в ЯЧате. После подтверждения ваш сервер получает результат проверки, который можно связать с заказом или другой операцией.

Важно: цифровой ID не является паролем. Доверяйте только подтверждённому ответу API, полученному вашим сервером. Секретный ключ нельзя размещать в браузере или мобильном клиенте.
Подтверждение в ЯЧате

Действие подтверждает сам владелец аккаунта через системный бот.

Проверка на сервере

Итоговая проверка выполняется только между сервером разработчика и ЯЧатом.

Отдельный идентификатор

Интеграция получает идентификатор, предназначенный именно для своего клиента.

Минимум данных

API возвращает только данные, необходимые для подтверждения операции.

Формат цифрового ID

Новые русскоязычные аккаунты получают ID с кириллическими буквами, например АБВ — 123. API также продолжает принимать ранее выданные латинские ID, например ABC — 123.

  • после нормализации ID всегда содержит ровно 6 символов: 2 буквы и 4 цифры либо 3 буквы и 3 цифры;
  • буквы одного ID должны принадлежать одному алфавиту: русскому или латинскому;
  • пробелы, обычный дефис и длинное тире при проверке игнорируются;
  • Ё нормализуется в Е;
  • не транслитерируйте кириллицу и не применяйте проверку, ограниченную диапазоном A-Z.

Порядок интеграции

  1. Зарегистрируйте приложение и получите параметры клиента.
  2. Создайте уникальную ссылку на будущую операцию — externalReference.
  3. Передайте цифровой ID в identity/initiate и попросите пользователя ввести код из ЯЧата.
  4. Подтвердите код через identity/confirm.
  5. Передайте результат на свой сервер и вызовите identity/consume до создания заказа или транзакции.

1. Начать подтверждение

POST/api/developer/v1/identity/initiate

{
  "clientId": "your-client-id",
  "digitalId": "АБВ — 123",
  "purpose": "Подтверждение заказа",
  "externalReference": "order-draft-7f30c4",
  "codeChallenge": "BASE64URL_SHA256_VERIFIER",
  "metadata": { "cartItems": 2 }
}

Ответ содержит challengeId, который понадобится на следующем шаге. Защитные параметры клиента формируются по выданной при регистрации спецификации.

2. Подтвердить код

POST/api/developer/v1/identity/confirm

{
  "clientId": "your-client-id",
  "challengeId": "1d32…",
  "code": "123456",
  "codeVerifier": "ORIGINAL_RANDOM_VERIFIER"
}

Успешный ответ содержит identityToken. Сразу передайте его на свой сервер и не сохраняйте в постоянном хранилище клиента.

3. Погасить токен на сервере

POST/api/developer/v1/identity/consume

const response = await fetch(
  "https://yachat.vercel.app/api/developer/v1/identity/consume",
  {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "X-YaChat-API-Key": process.env.YACHAT_API_KEY
    },
    body: JSON.stringify({
      clientId: process.env.YACHAT_CLIENT_ID,
      identityToken,
      codeVerifier,
      externalReference,
      metadata: { orderNumber: "A-1042" }
    })
  }
);

const proof = await response.json();
if (!response.ok || !proof.verified) throw new Error("Identity rejected");

// Свяжите подтверждение с операцией только после ответа verified: true.
await createOrder({ yachat: proof.identity, proofId: proof.transactionId });
{
  "verified": true,
  "transactionId": "57e5…",
  "externalReference": "order-draft-7f30c4",
  "identity": {
    "subject": "yachat_a1b2…",
    "displayName": "Имя пользователя",
    "digitalId": "АБВ — 123"
  }
}

Обработка ошибок

  • 400 — цифровой ID имеет неверный формат или параметры запроса неполны.
  • 401 — запрос не подтверждён или не авторизован.
  • 403 — клиенту не разрешено выполнить операцию.
  • 409 — операция уже завершена или конфликтует с другой проверкой.
  • 410 — подтверждение больше не действует.
  • 429 — запросов слишком много; повторите позже.

Не показывайте пользователю технические ответы целиком. Запишите код ошибки на сервере и предложите начать подтверждение заново.
