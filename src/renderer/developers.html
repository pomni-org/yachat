ЯЧат для разработчиков OpenAPI JSON
Digital ID API · v1

Подтверждённый пользователь ЯЧата в вашем сервисе

Пользователь вводит цифровой ID, получает одноразовый код в боте «Коды подтверждения», а ваш сервер погашает короткоживущий токен и получает стабильный идентификатор пользователя.

Важно: цифровой ID не является паролем. Не считайте человека подтверждённым, пока сервер не получил ответ verified: true от метода identity/consume.
Подтверждение в ЯЧате

Код видит только владелец аккаунта в системном боте «Коды подтверждения».

PKCE S256

Код и токен привязаны к вашему случайному verifier — перехваченного токена недостаточно.

Короткоживущий токен

Токен действует 5 минут. Повтор того же серверного запроса безопасен только с исходными verifier и externalReference.

Стабильный subject

Ваш сервер получает отдельный непрозрачный subject, постоянный для пары пользователь + клиент.

Порядок интеграции

  1. Зарегистрируйте клиент и получите clientId. Серверным клиентам также выдаётся X-YaChat-API-Key; публичным браузерным клиентам задаются разрешённые origins.
  2. Создайте случайный PKCE codeVerifier, вычислите S256 codeChallenge и сохраните verifier до конца операции.
  3. Передайте введённый ID в identity/initiate. Код появится у пользователя в ЯЧате.
  4. Отправьте код пользователя и verifier в identity/confirm. Полученный identityToken не сохраняйте в браузере надолго.
  5. Передайте токен, verifier и тот же externalReference на свой сервер. Сервер вызывает identity/consume до создания заказа или транзакции.

1. Начать подтверждение

POST/api/developer/v1/identity/initiate

{
  "clientId": "your-client-id",
  "digitalId": "ABC — 123",
  "purpose": "Подтверждение заказа",
  "externalReference": "order-draft-7f30c4",
  "codeChallenge": "BASE64URL_SHA256_VERIFIER",
  "metadata": { "cartItems": 2 }
}

Ответ содержит challengeId, канал yachat-codes и время истечения. Один пользователь может запросить не более пяти кодов от клиента за 15 минут.

2. Подтвердить код

POST/api/developer/v1/identity/confirm

{
  "clientId": "your-client-id",
  "challengeId": "1d32…",
  "code": "123456",
  "codeVerifier": "ORIGINAL_RANDOM_VERIFIER"
}

После пяти неверных попыток запрос блокируется. Успешный ответ содержит identityToken, действующий 5 минут.

3. Погасить токен на сервере

POST/api/developer/v1/identity/consume

const response = await fetch(
  "https://yachat.vercel.app/api/developer/v1/identity/consume",
  {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "X-YaChat-API-Key": process.env.YACHAT_API_KEY
    },
    body: JSON.stringify({
      clientId: process.env.YACHAT_CLIENT_ID,
      identityToken,
      codeVerifier,
      externalReference,
      metadata: { orderNumber: "A-1042" }
    })
  }
);

const proof = await response.json();
if (!response.ok || !proof.verified) throw new Error("Identity rejected");

// Сохраняйте proof.transactionId и proof.identity.subject с заказом.
await createOrder({ yachat: proof.identity, proofId: proof.transactionId });
{
  "verified": true,
  "transactionId": "57e5…",
  "externalReference": "order-draft-7f30c4",
  "identity": {
    "subject": "yachat_a1b2…",
    "displayName": "Имя пользователя",
    "digitalId": "ABC — 123",
    "rawDigitalId": "ABC123"
  }
}

Ошибки и повторные запросы

  • 401 — неверный код, API key, PKCE или identity token.
  • 403 — origin не зарегистрирован для публичного клиента.
  • 409 — challenge завершён или externalReference уже связан с другой проверкой.
  • 410 — код или identity token истёк.
  • 429 — превышен лимит запросов или неверных попыток.

Если ответ identity/consume потерялся из-за сети, сервер может повторить тот же запрос с теми же token, verifier и externalReference: API вернёт ту же подтверждённую транзакцию с replayed: true. Для новой проверки создайте новый PKCE verifier и новый challenge.
